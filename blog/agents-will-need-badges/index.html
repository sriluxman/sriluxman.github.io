Agents will need badges

Before agents work alongside operators and technicians, identity, traceability, and accountability must become first-class.

July 10, 2026

Software agents are becoming coworkers.

Today that happens mostly in development. Agents write code alongside developers, and the guardrails are arriving: defined identities for agents, scoped permissions, governance to keep a helpful system from doing something expensive. Getting this right is already challenging — and the stakes are still forgiving, because code gets reviewed before it ships.

The machine floor is next. It is not hard to see agents working alongside plant operators, service technicians, and supervisors: preparing a changeover, triaging an alarm, guiding a repair. Active coworkers in operations.

There, the identity question gets much louder. Operations cannot afford ambiguity about who did what. When something goes wrong in a plant, accountability is not a philosophical topic — it is an investigation. “The AI did it” is not an answer an auditor, an insurer, or a plant manager will accept.

Every human on a factory floor has an identity, a qualification, and someone accountable for their work. Agents will be held to the same standard: clear identity, scoped authority, and traceability for every action.

That is not bureaucracy slowing the future down. It is the deciding factor for whether agents get deployed in factory and field operations at all.
