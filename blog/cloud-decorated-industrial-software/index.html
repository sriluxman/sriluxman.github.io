Cloud-decorated industrial software

Containers and YAML can change the costume without changing how software behaves.

February 12, 2026

Industrial software wants to be cloud-native now.

That is not a bad thing. But much of it is not cloud-native yet. It is cloud-decorated.

The same old monolith, now running inside a container. The same manual recovery process, now described in YAML. The same release anxiety, now hidden behind Helm charts.

Cloud-native is not where software runs. It is how software behaves.

Can it recover without a human? Can it be observed without guessing? Can it be replaced without ceremony?

Industrial software has history. Machines live long. Customers do not upgrade because a conference slide said so. Legacy is often the rational outcome of real constraints.

So pretending is not cheating. Rehearsal is part of evolution.

The danger is forgetting that we are pretending. Then the costume becomes the strategy.
